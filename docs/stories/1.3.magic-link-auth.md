## Story

**As a** salon owner,
**I want** to sign in using my email via magic link (no password),
**so that** access is simple, frictionless, and I can start using SoftHair without memorizing credentials.

## Acceptance Criteria

1. Login screen accepts email (standard RFC-5322 format)
2. On submit, Supabase Auth sends magic link to the email via `signInWithOtp({ email })`
3. Click on email magic link redirects to `/auth/callback?code=...` which exchanges for session
4. Session valid for 30 days; auto-refresh when < 24h to expiry
5. Rate limit: max 3 magic link requests per email per 10 minutes (enforced by Supabase + app-level as backup)
6. Logout clears session cookies and redirects to login screen
7. Clear fallback UI if email delivery fails (error toast + "try again" button)
8. Schema: `public.users.email` becomes NOT NULL UNIQUE (primary identity for MVP); `phone_e164` becomes NULLABLE (Phase 2 WhatsApp integration reuse)
9. Trigger `on_auth_user_created` populates `public.users` from `auth.users.email` automatically on signup
10. Full flow passes E2E test (Playwright) using Supabase local + email provider mock OR with Supabase Auth test mode

## Dev Notes

### Context — scope change

Esta story foi **redesenhada 2026-04-21** como parte do scope change (WhatsApp removido do MVP). A versão anterior usava WhatsApp Business API via Meta Cloud. Agora usa **Supabase Auth nativo com email magic link** — muito mais simples, zero cost, zero dependency externa.

A migração para WhatsApp auth está **preservada** (documentada no Change Record) para Phase 2 quando retomarmos.

### Supabase Auth specifics

- `signInWithOtp({ email })` envia magic link automático
- Funciona em Supabase local (via mailpit interceptor) e remote (Supabase's SMTP ou custom SMTP)
- Sessão é gerenciada via httpOnly cookies pelo `@supabase/ssr` helper
- Não precisamos de tabelas `auth_otps` / `auth_otp_attempts` (estava na versão WhatsApp) — Supabase cuida internamente

### Integration with `@supabase/ssr`

Install no apps/web:
```bash
pnpm --filter @softhair/web add @supabase/ssr @supabase/supabase-js
```

Wrappers em `apps/web/lib/supabase/`:
- `server.ts` — `createServerClient` via `cookies()` from `next/headers`
- `client.ts` — `createBrowserClient` para Client Components
- `middleware.ts` — `createServerClient` via request/response cookies

Referência: https://supabase.com/docs/guides/auth/server-side/nextjs

### Schema changes (Task 1, 2)

**Migration 20260421000000:**
- `public.users.phone_e164`: NOT NULL → NULLABLE (Phase 2 reativa com WhatsApp)
- `public.users.email`: NULLABLE → NOT NULL UNIQUE (MVP primary identity)
- CHECK constraint para format básico de email (server-side validação via Zod é primary; CHECK é belt-and-suspenders)

**Migration 20260421000001:**
- Função `handle_new_auth_user()` — SECURITY DEFINER, popula public.users ao INSERT em auth.users
- Trigger `on_auth_user_created AFTER INSERT ON auth.users`
- DROP da policy `users_insert` (inserts agora são só via trigger)

### UX (from front-end-spec.md)

- Login screen: AuthLayout template, centered card, logo + título + single input + CTA
- Check-email screen: friendly + countdown timer para reenvio
- Error states: inline via `aria-invalid` + toast para erros de service
- WCAG AA: focus ring visível, keyboard nav completo, aria-labels

### Security

1. Email validation: Zod + PostgreSQL CHECK (defense in depth)
2. Rate limit: Supabase built-in (generous) + optional app-level (skip para MVP)
3. Session cookies: SameSite=Lax, Secure em prod, HttpOnly (Supabase SSR default)
4. Error messages generic (não vaza existence de email)
5. Redirect URL whitelisted no Supabase Dashboard (impede phishing via redirect)

### Testing

**Test file location**:
- Unit: co-located (`*.test.ts`)
- E2E: `apps/web/e2e/auth-magic-link.spec.ts`

**Framework**: Vitest (unit) + Playwright (e2e)

**Magic link em E2E (Supabase local)**:
```typescript
// Use service_role para gerar link sem envio de email real
const { data } = await admin.auth.admin.generateLink({
  type: 'magiclink',
  email: 'test@softhair.app',
});
// data.properties.action_link = URL para click
await page.goto(data.properties.action_link);
```

### Out of scope

- WhatsApp auth (Phase 2)
- SMS OTP (never — covered by email)
- Google/Facebook OAuth (Phase 2+)
- Password-based auth (rejected — friction)
- 2FA (Phase 2+ after validation with design-partners)
- Account recovery UI beyond "resend link" (Phase 2)

### Known gotchas

- Supabase local (via `supabase start`) usa **Mailpit** em `http://localhost:54324` para interceptar emails — fluxos de dev validados lá
- Supabase remote (softhair-dev, softhair-prod) usa SMTP padrão (limite gratuito ~4 emails/hour)
- Para produção com alto volume, configurar custom SMTP (Resend, Postmark) — **não crítico no MVP**
