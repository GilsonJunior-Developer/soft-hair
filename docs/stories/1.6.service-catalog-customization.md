## Story

**As a** salon owner,
**I want** to customize the service catalog (add, remove, adjust price and duration),
**so that** the salon reflects its operational reality.

## Acceptance Criteria

1. "Serviços" screen lists all customized services of the salon (seed from onboarding pre-loaded)
2. Add service: from global catalog (`service_catalog`) OR custom creation
3. Each service has: name, category, price (R$), duration (minutes, multiples of 15), custom commission (optional, overrides professional default)
4. Activate/deactivate service (inactive does not appear on public link)
5. Search + filter by category
6. Passes E2E test of service CRUD

- Variants: `selectable` (for customers picking service) vs `editable` (for owner customizing)

### Schema reference (from architecture.md §Data Models)

```
services (
  id UUID PK,
  salon_id UUID FK → salons,
  catalog_id UUID FK → service_catalog (nullable),
  name, category,
  duration_minutes INT CHECK (> 0 AND % 15 == 0),
  price_brl NUMERIC(10,2) CHECK (>= 0),
  commission_override_percent NUMERIC(5,2) CHECK (0-100),
  is_active BOOL,
  deleted_at TIMESTAMPTZ
)
```

### Catalog vs custom

- `catalog_id` IS NOT NULL → service derived from global catalog (editable price, not name/duration)
- `catalog_id` IS NULL → fully custom (all fields editable)
- On delete of catalog entry (superadmin rare action), customer's service survives (FK ON DELETE SET NULL)

### Testing

**Test file location**: `apps/web/e2e/service-crud.spec.ts`

**Standards**:

- Test edge cases: duration 14min (reject), duration 15min (accept), duration 480min (accept), duration 481min (reject)

### Out of scope

- Time-of-day pricing (peak vs off-peak) — Phase 2 (Dashboard de rentabilidade)
- Images per service — Phase 2
