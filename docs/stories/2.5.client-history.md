## Story

**As a** salon owner or receptionist,
**I want** to see each client's history (past + upcoming appointments + notes),
**so that** I can personalize service (remember what they did, preferences, allergies).

## Acceptance Criteria

1. `/clientes` lists all clients of the salon — sortable by last appointment, name, total visits (count)
2. Search by name OR phone — debounced, server-driven
3. Client detail page `/clientes/[id]` shows: basic data (name, phone, email), last 10 appointments (service, professional, date, notes), upcoming scheduled appointments, placeholder credit-balance (R$ 0,00 — Epic 5)
4. "Observações do atendimento" editable per appointment post-service (free text stored on `appointments.notes`)
5. Soft delete of client with confirmation dialog (sets `deleted_at`)
6. Performance: list with 1000+ clients renders in ≤ 1s with pagination (page size 25)

### Query for list with stats

```sql
SELECT
  c.id, c.name, c.phone_e164, c.email, c.created_at,
  COUNT(a.id) FILTER (WHERE a.status = 'COMPLETED') AS visit_count,
  MAX(a.scheduled_at) FILTER (WHERE a.status = 'COMPLETED') AS last_appointment_at
FROM public.clients c
LEFT JOIN public.appointments a ON a.client_id = c.id
WHERE c.salon_id = <current> AND c.deleted_at IS NULL
GROUP BY c.id
ORDER BY last_appointment_at DESC NULLS LAST
LIMIT 25 OFFSET ?;
```

Consider materializing once it starts hurting — for MVP this is fine up to a few thousand clients.
