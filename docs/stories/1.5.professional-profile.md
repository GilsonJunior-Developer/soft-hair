## Story

**As a** salon owner,
**I want** to register professionals with name, commission, and availability,
**so that** appointments can be assigned to them.

## Acceptance Criteria

1. "Profissionais" screen listing registered professionals (empty at start, with CTA "Adicionar profissional")
2. Registration form: name, optional photo (uploaded to Supabase Storage), specialties (multi-select from catalog), working hours (days + windows), commission rule (% fixed OR table per service)
3. Edit and remove (soft delete) supported
4. Inactive professional does not appear on public agenda
5. Salon can have 1-20 professionals (limit validation)
6. Photo automatically resized to ≤ 500KB
7. Passes E2E test of professional CRUD

### Storage bucket RLS (critical security)

```sql
-- Bucket 'professional-photos' RLS (via Supabase Dashboard or CLI)
CREATE POLICY "Users can upload to their salon" ON storage.objects
  FOR INSERT WITH CHECK (
    bucket_id = 'professional-photos'
    AND (storage.foldername(name))[1]::UUID IN (SELECT public.current_user_salon_ids())
  );

CREATE POLICY "Public can read photos" ON storage.objects
  FOR SELECT USING (bucket_id = 'professional-photos');
```
