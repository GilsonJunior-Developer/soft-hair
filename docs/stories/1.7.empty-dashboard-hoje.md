## Story

**As a** salon owner,
**I want** to see a clear initial screen showing what will happen today,
**so that** I have immediate context when opening the system.

## Acceptance Criteria

1. Dashboard displays: today's date, next 3 appointments (empty now, friendly placeholder "Nenhum agendamento hoje ainda"), 3 placeholder metrics (appointments today, daily revenue, occupancy rate — all zero)
2. Header with logo, salon name, user avatar + logout menu
3. Lateral navigation (desktop) or bottom-nav (mobile) with: Hoje, Agenda, Profissionais, Serviços, Clientes (placeholder), Financeiro (placeholder), Configurações
4. Responsive mobile-first: layout adapts at ≤ 375px
5. PWA manifest configured (installable)
6. Accessibility: keyboard navigation + ARIA labels validated with axe-core
7. Passes E2E rendering test in Chrome + Safari iOS (via Playwright)

### PWA gotchas

- iOS Safari: requires `apple-mobile-web-app-capable` meta + specific icon sizes
- Android Chrome: install prompt controlled by `beforeinstallprompt` event — can be custom UI (front-end-spec mentions custom prompt after 3 visits)
- Service worker: start simple (network-first for pages, cache-first for assets); can optimize in Phase 2

### Font loading strategy (Tailwind v4 + next/font)

```tsx
// apps/web/app/layout.tsx
import { Inter, Playfair_Display } from 'next/font/google';

const inter = Inter({ subsets: ['latin'], variable: '--font-sans' });
const playfair = Playfair_Display({ subsets: ['latin'], variable: '--font-display' });

export default function RootLayout({ children }) {
  return (
    <html lang="pt-BR" className={`${inter.variable} ${playfair.variable}`}>
      <body>{children}</body>
    </html>
  );
}
```

In `globals.css`:
```css
@theme {
  --font-sans: var(--font-sans);
  --font-display: var(--font-display);
}
```
