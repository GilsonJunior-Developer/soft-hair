## Story

**As a** salon client (end user),
**I want** to book a service through the public link by providing my name + email + phone (E.164),
**so that** booking takes ≤ 60 seconds on mobile without installing any app.

## Acceptance Criteria

1. 3-step flow on `/(public)/{salon}/{professional}/book`: (a) escolher serviço, (b) escolher data+horário disponível, (c) informar nome + email + telefone → confirmar
2. Step (b) shows **only truly available slots** — respects `professionals.working_hours_jsonb`, service `duration_minutes`, and existing appointments (status IN `PENDING_CONFIRMATION`/`CONFIRMED`)
3. Phone validation: Brazilian E.164 format via regex + optional Zod transform (`+55 (11) 9xxxx-xxxx` masked input, stored normalized)
4. Email validation: RFC-compliant + MX-relaxed; required field
5. LGPD consent checkbox: **required**, non-prechecked, labeled "Concordo que meus dados sejam usados para confirmar este agendamento e manter histórico no salão"
6. On confirm, appointment persists with `status = 'PENDING_CONFIRMATION'`; client upserted into `clients` (phone+email unique per salon)
7. Success screen with: resumo (serviço, profissional, data/hora, preço), "Um email de confirmação foi enviado para {email}" (Story 2.7 entrega o link), e botão "Compartilhar" (Web Share API) para o cliente salvar o resumo
8. Full flow completes in ≤ 60s on a clean run (Playwright timing, deferred)
9. Analytics event `booking_completed` fires (console log acceptable for MVP; add real analytics later)

### Availability algorithm sketch

```ts
// conceptual pseudocode
for each day in [from, to):
  workingRanges = expand(professional.working_hours_jsonb[dayOfWeek])
  booked = appointments.where(...).map(a => [a.scheduled_at, a.ends_at])
  freeRanges = subtract(workingRanges, booked)
  for start of freeRanges in 15-min steps:
    if start + service.duration_minutes <= range.end: emit start
```

### Email transactional delivery

Supabase free tier doesn't include SMTP for custom templates. Options:
1. **Resend** — generous free tier (100 emails/day), simple REST API, trivial to add

Recommendation: Resend. Add `RESEND_API_KEY` to env, `apps/web/lib/email.ts::sendBookingConfirmation`. If key missing, log + skip (don't break flow).
